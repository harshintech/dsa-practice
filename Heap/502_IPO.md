# LeetCode 502 - 🏗 IPO – Find Maximized Capital

---
## [Interview-Preparation GitHub Repo Contribute Now](https://github.com/harshintech/Interview-Preparation.git)


## 📌 Problem

You are given:

- `k` → maximum number of projects you can select  
- `w` → initial capital  
- `profits[]` → profit of each project  
- `capital[]` → minimum capital required for each project  

Each project:

- Requires certain capital to start  
- Gives profit after completion  

Return the **maximum capital** after selecting at most `k` projects.

---

## 🧩 Example

Input:
```
k = 2
w = 0
profits = [1,2,3]
capital = [0,1,1]
```

Output:
```
4
```

Explanation:

Start with 0:

- Can pick project 0 → profit 1 → capital = 1
- Now can pick project 2 → profit 3 → capital = 4

---

# 🚀 Approach (Greedy + Sorting + Max Heap)

---

## 🔑 Key Idea

1️⃣ Sort projects by required capital  
2️⃣ Use a max heap to always pick the highest profit among affordable projects  
3️⃣ Repeat up to k times  

This ensures:

```
At each step → choose the most profitable available project
```

---

# 💻 Java Code

```java
class Solution {

    private static class Project {
        int capital;
        int profit;

        Project(int capital, int profit) {
            this.capital = capital;
            this.profit = profit;
        }
    }

    public int findMaximizedCapital(int k, int w,
                                    int[] profits, int[] capital) {

        int n = profits.length;
        List<Project> projects = new ArrayList<>();

        for (int i = 0; i < n; i++) {
            projects.add(new Project(capital[i], profits[i]));
        }

        // Sort by required capital
        Collections.sort(projects,
            (a, b) -> a.capital - b.capital);

        // Max heap for profits
        PriorityQueue<Integer> maxHeap =
            new PriorityQueue<>((x, y) -> y - x);

        int i = 0;

        for (int j = 0; j < k; j++) {

            while (i < n && projects.get(i).capital <= w) {
                maxHeap.add(projects.get(i).profit);
                i++;
            }

            if (maxHeap.isEmpty()) {
                break;
            }

            w += maxHeap.poll();
        }

        return w;
    }
}
```

---

# 🎯 Why Sorting by Capital?

We need to know:

```
Which projects are affordable at current capital?
```

Sorting ensures:

We can iterate once and push affordable projects into heap.

---

# 🎯 Why Max Heap?

Among affordable projects:

We always choose the one with:

```
Maximum profit
```

Greedy choice.
